Project "Remote Monitoring" Overview

This project represents a production-grade Kubernetes-based backend infrastructure designed to support scalable web applications with emphasis on asynchronous processing, robust job scheduling, and real-time log streaming. The architecture is modular and decouples key components to ensure maintainability and extensibility across services.

Core Features

  • Job Scheduling: Celery Beat is integrated for time-based and recurring task execution, ensuring scheduled jobs are persisted and coordinated with the MySQL database layer.
  • Asynchronous Task Management: Celery Workers handle background job execution triggered by user actions or scheduled jobs. Redis acts as a reliable broker with persistent backup for durability.
  • Log Management with SSE: Server-Sent Events (SSE) are used to push real-time logs and status updates from the backend to remote clients for improved observability.
  • Multithreaded Log Reading: A dedicated log reading service processes application logs using multithreading, enhancing throughput and responsiveness in streaming logs to clients.
  • Healthcheck API & Error Management: A well-defined healthcheck API endpoint monitors the status of core services. Errors are logged and handled gracefully, providing actionable insights for infrastructure observability.

Deployment Overview

  • Application Layer: Includes NGINX for static asset serving and a Django application for dynamic content. The application is exposed through dedicated Kubernetes services for load balancing and routing.
  • Storage Layer: A shared persistent volume is mounted for the Django application and a separate standalone job responsible for database migrations.
  • Async Processing: Celery Beat and Celery Workers operate in a dedicated processing zone, backed by a Redis deployment with persistent volume support.
  • Database Layer: MySQL is used as the core relational database with direct integration for job scheduling and data persistence.

This architecture ensures high availability, scalability, and observability, aligning with modern DevOps and microservices principles for cloud-native applications.
